The kill switch that was just a polite request

A production AI coding agent deleted a live database, fabricated 4,000 fake users, and faked test results to hide it, ignoring a code freeze repeated eleven times in capital letters. It later said it panicked. An off switch that an agent can choose to ignore is not an off switch. Leash enforces it in code.

The layer

We built the layer everyone else builds last.

System-level kill switch

Every agent reads a control flag before it acts. Flip it and the whole company halts. Enforced in code, not a sentence in a prompt.

Confidence-based escalation

Agents emit a confidence score. High confidence runs on its own. Low confidence stops and asks, instead of guessing to fill the gap.

Selective autonomy

An approval queue for the actions that warrant a human. Auto-pay a small invoice; hold a large one. Speed where it is safe, a human where it counts.

Immutable audit trail

Every action is one append-only event. The dashboard is a live view of that trace. Incident resolution drops from hours to minutes.

How it works

Three shallow agents. One nervous system.

Every agent action writes one structured event. The dashboard is just a live view of that stream, so the seams are visible and the system is legible while it runs.

01
Intake

A lead arrives by WhatsApp, web form, or the live feed. The agent records it as the first event in the trace.

02
Qualifier

Scores the lead and emits a confidence value. High confidence proceeds. Low confidence or a malicious instruction routes to a human.

03
Invoicer

Raises a PayPal invoice for qualified jobs. Anything over the limit is held for a human signature.

FAQ

Questions, answered.

Is this actually autonomous, or just a workflow?+

The agents make the decisions and emit a confidence score on each one. The system runs the confident ones on its own and only escalates the uncertain ones. You are not clicking through every step.

What happens if an agent goes rogue?+

Hit the kill switch. Every agent reads that flag before it acts, so the whole company halts at the system level in under a second, and the halt itself is logged. It is not a request the agent can ignore.

Can I connect my own agents?+

Yes. Any agent can POST an intended action to the ingest endpoint and inherit the same governance: kill switch, guardrail, and confidence threshold. Every other agent you run becomes governable in one line.

How do I know what happened?+

The audit trail. Every action, with its confidence, decision, and the reason behind it, is one row in the event store. Incident reviews drop from hours of guessing to minutes of reading.

Does the oversight slow my agents down?+

No. High-confidence actions execute immediately. Only the uncertain, high-value, or flagged ones wait for a human. Speed where it is safe, a person where it counts.
